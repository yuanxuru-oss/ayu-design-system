# Scene: 教程型 / 说明型 / 知识页

> 适用于教程页、readme 式说明页、玩法介绍页、概念拆解页。目标是像小岛知识布告栏，不像文档站或课程后台。

---

## 核心气质

- 像“住在岛上的人认真做的一页说明”
- 有引导感，但不官方
- 有系统感，但不工程化
- 有生活痕迹，不像模板网页

---

## 视觉基调

- 外层优先保留森林绿叶片纹理背景
- 内容主体放在暖米/米白阅读层上
- 标题像木牌、护照标签、任务标题
- 局部可加入叶片、胶带、图章、圆角贴纸

---

## 布局优先级

优先从 `references/layouts.md` 里使用：

1. `Leaf Board Banner`
2. `Board + Sidebar`
3. `Bulletin Stack`
4. `Guide Path Steps`
5. `Module Grid`

页面建议节奏：

1. 开场欢迎板
2. 教程目录 / 任务概览
3. 步骤说明块
4. 关键提示或补充便签
5. 结尾总结 / 下一步入口

---

## 组件偏好

- `Board Panel`
- `Memo Card`
- `Status Panel`
- `Module Entry Card`

重点：
- 每一块内容都要像系统中的一个小部件
- 不要出现纯文字直接裸排的默认 section

---

## 内容组织方式

### 开场区
- 放欢迎语、主题、适用人群、阅读方式
- 可以有头像或角色标签

### 步骤区
- 更像任务链或玩法说明
- 3 到 6 步最佳
- 每步可有一个状态 icon 或小贴纸

### 提示区
- 用便签或提示卡承载“注意事项 / 小诀窍 / 常见误区”
- 不要用默认引用框

### 收尾区
- 给出“你接下来可以去哪里”
- 可以挂一个功能入口卡，而不是只放一句结束语

---

## 语气

- 像陪人一起上手，不像讲课
- 解释清楚，但不过度“老师感”
- 能说人话就不堆术语

---

## 禁忌

- 不要使用 Esther 式衬线 editorial 标题体系
- 不要靠大编号、时间线、深色大面板制造教程节奏
- 不要做成 SaaS help center
- 不要让页面像普通 README 截图
